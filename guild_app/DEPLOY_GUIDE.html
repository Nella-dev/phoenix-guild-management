⚡ 배포 방법 (Firebase Hosting)

Step 1. Node.js 및 Firebase CLI 설치 
npm install -g firebase-tools
Step 2. Firebase 로그인 
firebase login

→ 브라우저에서 Google 계정 인증

Step 3. 프로젝트 디렉토리에서 배포 
cd /path/to/roider-project
firebase deploy --only hosting
Step 4. 배포 완료 URL 
https://roider-guild-management.web.app

📐 Firestore 문서 구조

shared_formations/{docId} {
  title:         "진형 제목",
  description:   "전략 설명",
  arena:         "Arena I",
  formationMode: 1~5,         // 진형 수
  tiles:         [45개 숫자], // 헥스 그리드 타일
  teams:         { ... },     // 팀 데이터 (옵션)
  authorId:      "Firebase UID",
  authorName:    "닉네임",
  createdAt:     Timestamp,
  likes:         0
}

🔒 Firestore Security Rules (현재 설정)

match /shared_formations/{docId} {
  allow read:          if request.auth != null;  // 로그인 멤버만 읽기
  allow create:        if request.auth != null;  // 로그인 멤버만 작성
  allow update,delete: if request.auth != null
    && request.auth.uid == resource.data.authorId; // 본인만 수정/삭제
}
💡 참고: Firestore rules는 로그인한 멤버만 읽기/쓰기 가능합니다. 비로그인 상태에서는 편성이 보이지 않으며, 이것은 의도된 보안 설정입니다.
